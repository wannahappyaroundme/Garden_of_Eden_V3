## 약점 (개선 필요)

### 1. 첫 사용자 경험 (Onboarding)
- ❌ 온보딩이 작동하지 않음 (라우팅 오류)
- ⚠️ 3초 후 자동 전환 (너무 빠름)
- ✅ 좋은 가이드 질문들

**개선**: 라우팅 수정 + "시작하기" 버튼 추가

### 2. 발견 가능성 (Discoverability)
- ❌ 단축키 존재를 모름 (⅓ 사용자가 미발견)
- ❌ 음성 입력 기능 가시성 낮음
- ❌ 피드백 버튼 (👍/👎) 오른쪽에 숨김
- ⚠️ 제안 프롬프트 빈 상태에만 표시

**개선**: 온보딩에서 기능 소개 + 우측 사이드바

### 3. 피드백/상태
- ❌ 설정 저장 확인 부족
- ❌ 에러 메시지 실행 방법 없음
- ⚠️ 로딩 중 시간 정보 없음
- ✅ 음성 입력 시각화는 좋음

**개선**: Toast 알림 + 에러별 가이드

### 4. 정보 구조
- ⚠️ "앱 설정"과 "AI 성격 설정" 구분 모호
- ⚠️ 아코디언 첫 번째만 열림 (이유 불명확)
- ⚠️ 대화 정렬 옵션 없음
- ✅ 검색 기능은 있음

**개선**: 탭으로 분리 + 정렬 옵션 추가

### 5. 일관성
- ⚠️ SVG 직접 정의 + lucide-react 혼재
- ⚠️ 버튼 스타일 계층 불명확
- ⚠️ 모달 닫기 방식 불일치
- ✅ 한국어 용어 일관성 좋음

**개선**: 아이콘 라이브러리 통일 + 버튼 계층 정의

## 사용자 시나리오별 분석

### 시나리오 1: 신규 사용자 (처음 실행)
```
1. 앱 실행
   └─ 🔴 온보딩 페이지인데 라우팅 오류 → 막힘
   
예상 결과: 첫 번째 장애벽에서 좌절
```

**개선 후**:
```
1. 앱 실행
   └─ ✅ 온보딩 페이지 표시
2. 질문 답변
   └─ ✅ "시작하기" 버튼으로 진행
3. 채팅 화면 진입
   └─ ✅ 제안 프롬프트 표시
4. 첫 메시지 전송
   └─ ✅ AI 응답 표시
```

### 시나리오 2: 일상 사용자
```
1. 채팅 입력
   ├─ ⚠️ Shift+Enter 방법 모름
   └─ 가이드 텍스트가 맨 아래에 있어서 못 봄
   
2. 주제 변경 원할 때
   └─ ❌ 제안 프롬프트가 없어짐
   └─ 다시 새 대화 시작해야 함

3. 설정 변경
   ├─ 저장했는데 확인이 안 됨
   └─ 실제 저장되었는지 불안함
```

### 시나리오 3: 전력 사용자
```
1. 단축키로 효율적으로 작업
   ├─ Cmd+K: 입력창 포커스 ✅
   ├─ Cmd+,: 설정 열기 ✅
   └─ ?: 단축키 도움말 ✅
   
2. 성격 커스터마이징
   ├─ 아코디언 첫 그룹이 열려있음 (왜?)
   └─ 다른 그룹은 안 열림 (불편)
   
3. 대화 관리
   ├─ 대화 검색 ✅
   └─ 정렬 옵션이 없음 (오래된순 원함)
```

## 개선 로드맵

```
Mon: 온보딩 라우팅 수정
Tue: 설정 저장 Toast 추가
Wed: 에러 메시지 가이드
Thu: 테스트 & 검증
```

- 긴급 (Week 1): 5-6일
- 중요 (Week 2): 8-10일
- 개선 (Week 3): 8-10일
- 총 1개월

**Q: 우선순위는?**
A: 
1. 온보딩 라우팅 (단 1개)

4. 일관성 (디자인 시스템)
