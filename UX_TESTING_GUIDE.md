## 테스트 개요

### 목표
Garden of Eden V3의 UX가 사용자 친화적이고 직관적인지 검증

### 테스트 환경
- **OS**: macOS 14+ 또는 Windows 11
- **RAM**: 최소 16GB (권장 24GB)
- **AI 모델**: Qwen 2.5 14B (9.0GB), Whisper, LLaVA
- **브라우저 DevTools**: 성능 모니터링용

### 테스트 기간
- **초기 테스트**: 30분 (핵심 플로우)
- **전체 테스트**: 2시간 (모든 기능)

---

## 테스트 준비

### 1. 환경 설정

```bash
# 1. 프로젝트 클론 및 의존성 설치
git clone https://github.com/wannahappyaroundme/Garden_of_Eden_V3.git
cd Garden_of_Eden_V3
npm install

# 2. Ollama 설치 및 모델 다운로드
brew install ollama  # macOS
ollama pull qwen2.5:14b

# 3. 개발 모드 실행
npm run dev
```

### 2. 초기 상태 확인

- [ ] 앱이 정상적으로 실행되는가?
- [ ] AI 모델이 로드되는가? (첫 실행 시 6-10초 소요)
- [ ] 메모리 사용량이 정상 범위인가? (10-14GB)

## 핵심 UX 플로우 테스트

### 플로우 1: 첫 사용자 온보딩 (신규 설치)

**시나리오**: 앱을 처음 설치한 사용자

1. **Welcome Screen** (예정)
   - [ ] 환영 메시지가 명확한가?
   - [ ] 다음 단계가 직관적으로 안내되는가?

2. **AI 모델 다운로드** (자동)
   - [ ] 다운로드 진행률이 표시되는가?
   - [ ] 예상 시간이 표시되는가?
   - [ ] 일시정지/재개 기능이 작동하는가?

3. **Persona 설정**
   - [ ] Settings → AI 성격 탭 접근
   - [ ] 6개 프리셋 중 하나 선택 (예: Friendly)
   - [ ] 개별 파라미터 조정 (formality, humor, verbosity)
   - [ ] 저장 버튼이 명확한가?
   - [ ] 저장 성공 토스트가 표시되는가?

4. **첫 대화**
   - [ ] 채팅 입력창이 즉시 눈에 띄는가?
   - [ ] "안녕하세요" 입력 후 전송
   - [ ] AI 응답이 2-5초 내에 도착하는가?
   - [ ] 응답이 선택한 persona를 반영하는가?

**예상 시간**: 5-10분

---

### 플로우 2: 일상적인 대화

**시나리오**: 코딩 도움 요청

1. **질문 입력**
   ```
   한국어로 대답해줘: React에서 useState를 사용하는 방법은?
   ```

2. **확인 사항**
   - [ ] Typing indicator가 표시되는가?
   - [ ] 응답이 스트리밍으로 표시되는가?
   - [ ] 코드 블록이 올바르게 렌더링되는가?
   - [ ] 응답이 한국어로 작성되었는가?
   - [ ] 응답 품질이 만족스러운가?

3. **후속 질문**
   ```
   useEffect는 어떻게 쓰는 거야?
   ```
   - [ ] 컨텍스트가 유지되는가? (이전 대화 참조)
   - [ ] 응답 속도가 일관적인가?

**예상 시간**: 5분

---

### 플로우 3: 설정 변경 및 적용

**시나리오**: Persona 파라미터 조정

1. **Settings 접근**
   - [ ] 우측 상단 ⚙️ 버튼 클릭
   - [ ] 설정 페이지가 즉시 로드되는가?

2. **AI 성격 탭**
   - [ ] 파라미터 그룹이 아코디언으로 정리되어 있는가?
   - [ ] 첫 번째 그룹(Communication Style)이 기본 열려 있는가?
   - [ ] Formality 슬라이더 조정 (5 → 8)
   - [ ] Emoji Usage 슬라이더 조정 (5 → 2)

3. **Preview Panel**
   - [ ] 우측에 미리보기가 표시되는가?
   - [ ] 조정한 값이 실시간으로 반영되는가?
   - [ ] 예시 응답이 변경된 스타일을 반영하는가?

4. **저장 및 확인**
   - [ ] "저장" 버튼 클릭
   - [ ] 성공 토스트가 표시되는가? ("설정이 저장되었습니다")
   - [ ] 채팅으로 돌아가기
   - [ ] 새 질문: "안녕하세요" → 응답이 변경된 스타일을 반영하는가?

**예상 시간**: 5분

## 기능별 테스트 시나리오

### 1. 채팅 인터페이스

#### 1.1 메시지 입력

- [ ] **단축키**: `Cmd+K` (macOS) / `Ctrl+K` (Windows)로 포커스 이동
- [ ] **Enter**: 메시지 전송
- [ ] **Shift+Enter**: 줄바꿈
- [ ] **긴 메시지** (500자 이상): 스크롤 가능한가?
- [ ] **특수문자**: 이모지, 한글, 영어, 코드 블록이 올바르게 표시되는가?

#### 1.2 메시지 표시

- [ ] **User 메시지**: 우측 정렬, 파란색 배경
- [ ] **AI 메시지**: 좌측 정렬, 회색 배경
- [ ] **타임스탬프**: 각 메시지에 표시되는가?
- [ ] **코드 블록**:
  ```python
  def hello():
      print("Hello World")
  ```
  - Syntax highlighting이 적용되는가?
  - Copy 버튼이 작동하는가?

#### 1.3 스크롤 및 히스토리

- [ ] **Auto-scroll**: 새 메시지 도착 시 자동 스크롤
- [ ] **Manual scroll**: 위로 스크롤 시 자동 스크롤 비활성화
- [ ] **History 로드**: 이전 대화 내역이 유지되는가?
- [ ] **검색** (예정): 대화 내역 검색 기능

---

### 2. Suggestions Panel

#### 2.1 카테고리별 제안

**테스트 단계**:
1. 채팅 입력창 하단의 "💡 추천" 영역 확인
2. 각 카테고리 클릭:
   - [ ] 💬 **Casual**: "오늘 날씨 어때?"
   - [ ] 💻 **Code**: "리스트를 역순으로 정렬하는 파이썬 함수"
   - [ ] 📝 **Writing**: "이메일 작성 도와줘"
   - [ ] 🎨 **Creative**: "짧은 시 하나 지어줘"

3. **동작 확인**:
   - [ ] 클릭 시 입력창에 텍스트가 자동으로 채워지는가?
   - [ ] 즉시 전송되는가? (아니면 수정 가능한가?)
   - [ ] 응답이 카테고리에 맞는가?

#### 2.2 컨텍스트 기반 제안 (예정)

- [ ] 대화 흐름에 따라 제안이 변경되는가?
- [ ] 최근 대화 주제와 관련된 제안이 표시되는가?

### 3. Toast Notifications

#### 3.1 Success Toast

**트리거**: Settings에서 "저장" 버튼 클릭

- [ ] 녹색 체크마크 아이콘
- [ ] "설정이 저장되었습니다" 메시지
- [ ] "변경사항이 즉시 적용됩니다" 설명
- [ ] 3초 후 자동 사라짐
- [ ] X 버튼으로 수동 닫기 가능

#### 3.2 Error Toast

**트리거**: 네트워크 오류 시뮬레이션 (개발자 도구에서)

- [ ] 빨간색 경고 아이콘
- [ ] "설정 저장 실패" 메시지
- [ ] "네트워크 연결을 확인하거나 다시 시도해주세요" 설명
- [ ] 5초 후 자동 사라짐

#### 3.3 Info Toast

**트리거**: Settings에서 "초기화" 버튼 클릭 후 확인

- [ ] 파란색 정보 아이콘
- [ ] "설정이 초기화되었습니다" 메시지
- [ ] "저장 버튼을 눌러 변경사항을 적용하세요" 설명

### 4. Settings 페이지

#### 4.1 AI 성격 탭

**파라미터 그룹 (6개)**:

1. **Communication Style** (기본 열림)
   - [ ] Formality (격식 수준)
   - [ ] Humor (유머 사용)
   - [ ] Emoji Usage (이모지 사용)
   - [ ] Verbosity (말수)

2. **Emotional Intelligence**
   - [ ] Empathy (공감 능력)
   - [ ] Encouragement (격려)
   - [ ] Patience (인내심)

3. **Technical Depth**
   - [ ] Technicality (기술 수준)
   - [ ] Code Language Preference

4. **Creativity & Proactivity**
   - [ ] Creativity (창의성)
   - [ ] Proactivity (능동성)

5. **Reasoning & Context**
   - [ ] Reasoning Depth (추론 깊이)
   - [ ] Context Awareness (맥락 인식)

6. **Korean Language**
   - [ ] Formality Honorifics (존댓말 수준)
   - [ ] Language Preference

**테스트**:
- [ ] 각 그룹 클릭 시 부드럽게 열리고 닫히는가?
- [ ] 슬라이더가 부드럽게 움직이는가?
- [ ] Tooltip이 표시되는가? (각 파라미터 설명)
- [ ] 값 변경 시 Preview Panel이 업데이트되는가?

#### 4.2 앱 설정 탭

- [ ] **다크 모드**: 토글 스위치가 작동하는가?
- [ ] **언어 선택**: 한국어 ↔ English 변경 시 UI가 즉시 변경되는가?
- [ ] **코드 언어**: TypeScript, Python, Rust 등 선택 가능

#### 4.3 정보 탭

- [ ] **버전 정보**: 1.0.0 표시
- [ ] **플랫폼**: darwin (macOS) 또는 win32 (Windows)
- [ ] **AI 모델**: Qwen 2.5 14B Instruct
- [ ] **시스템 정보**:
  - 100% 로컬 실행 ✓
  - 메모리 사용량 표시
- [ ] **도움말 링크**: 클릭 시 외부 브라우저 열림

### 5. 에러 처리

#### 5.1 AI 응답 실패

**시뮬레이션**: Ollama 서비스 중단

```bash
# Terminal에서 Ollama 중지
pkill ollama
```

**확인 사항**:
- [ ] 명확한 에러 메시지가 표시되는가?
- [ ] "AI 모델과 연결할 수 없습니다. Ollama가 실행 중인지 확인하세요." 같은 actionable한 메시지
- [ ] 재시도 버튼이 있는가?
- [ ] 앱이 크래시하지 않는가?

#### 5.2 네트워크 오류 (예정 - 외부 API 통합 시)

- [ ] Calendar API 연결 실패 시 명확한 안내
- [ ] Webhook 전송 실패 시 재시도 옵션

## 성능 및 반응성 테스트

### 1. 응답 속도

| 작업 | 목표 | 측정값 | 통과 여부 |
|------|------|--------|-----------|
| 첫 메시지 전송 | <5s | _____s | ☐ |
| 후속 메시지 전송 | <3s | _____s | ☐ |
| Settings 페이지 열기 | <200ms | _____ms | ☐ |
| Persona 저장 | <500ms | _____ms | ☐ |
| 다크모드 토글 | <100ms | _____ms | ☐ |

### 2. 메모리 사용량

```bash
# macOS
Activity Monitor → Garden of Eden V3 확인

# Windows
Task Manager → Details → Garden of Eden V3.exe
```

- [ ] **Idle**: <11GB (AI 모델 로드 후)
- [ ] **대화 중**: <14GB
- [ ] **설정 변경 시**: <14GB
- [ ] **메모리 누수 없음**: 10분간 사용 후 메모리가 계속 증가하지 않는가?

### 3. UI 프레임률

**Chrome DevTools 사용**:
1. `Cmd+Option+I` → Performance 탭
2. Record 시작
3. 30초간 일반 사용 (채팅, 스크롤, 설정 변경)
4. Record 중지

**확인**:
- [ ] FPS가 대부분 60에 가까운가?
- [ ] Dropped frames < 5%
- [ ] Long tasks (>50ms) 없음

## 접근성 테스트

### 1. 키보드 네비게이션

- [ ] **Tab**: 포커스 이동이 논리적 순서인가?
- [ ] **Shift+Tab**: 역순 이동
- [ ] **Enter**: 버튼 활성화, 메시지 전송
- [ ] **Escape**: 모달/설정 닫기
- [ ] **Cmd+K**: 채팅 입력창 포커스
- [ ] **포커스 표시**: 현재 포커스된 요소가 명확히 보이는가?

### 2. 스크린 리더 (선택 사항)

**macOS VoiceOver**:
```bash
Cmd+F5  # VoiceOver 활성화
```

- [ ] 버튼 이름이 명확한가? ("설정", "저장", "닫기" 등)
- [ ] 입력창에 aria-label이 있는가?
- [ ] 에러 메시지가 읽히는가?

### 3. 색상 대비

- [ ] 텍스트가 배경과 충분한 대비를 이루는가? (WCAG AA: 4.5:1)
- [ ] 다크모드에서도 가독성이 좋은가?
- [ ] 색맹 사용자도 정보를 구분할 수 있는가? (색상에만 의존하지 않음)

## 부록: 유용한 단축키

### 앱 내 단축키

| 단축키 | 기능 |
|--------|------|
| `Cmd+K` / `Ctrl+K` | 채팅 입력창 포커스 |
| `Cmd+,` / `Ctrl+,` | 설정 열기 (예정) |
| `Cmd+N` / `Ctrl+N` | 새 대화 시작 (예정) |
| `Escape` | 모달/설정 닫기 |
| `Enter` | 메시지 전송 |
| `Shift+Enter` | 줄바꿈 |

### 개발자 도구

| 단축키 | 기능 |
|--------|------|
| `Cmd+Option+I` / `F12` | DevTools 열기 |
| `Cmd+R` / `Ctrl+R` | 새로고침 |
| `Cmd+Shift+R` / `Ctrl+Shift+R` | 캐시 무시 새로고침 |
