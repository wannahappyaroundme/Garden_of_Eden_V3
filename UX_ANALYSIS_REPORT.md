## 1. 사용자 여정 (User Journey) 분석

### 1.1 첫 실행 → 온보딩 → 첫 대화 → 지속 사용

#### 문제점 발견

**A. 온보딩 페이지 - 라우팅 불일치**
- 파일: `Onboarding.tsx`에서 `useNavigate()` 사용
- 문제: App.tsx는 상태 기반 라우팅(Chat/Settings)을 사용하는데, Onboarding에서 React Router 사용
- **영향**: 온보딩이 실제로 작동하지 않을 수 있음 (라우터 설정 필요)
- **사용자 경험**: 첫 사용자가 온보딩을 완료할 수 없음 = 치명적 문제

**B. 온보딩 스킵 경로 부족**
- 온보딩 이슈로 인해 새 사용자가 바로 Chat으로 가야 함
- 현재: "나중에 할게요" 버튼이 있지만 라우터 문제로 작동 불확실
- **개선**: 온보딩을 선택적으로 완전히 건너뛸 수 있는 명확한 경로 필요

**C. 온보딩에서 채팅으로의 부자연스러운 전환**
- 온보딩이 완료되면 자동으로 3초 후 Chat으로 이동 (라인 193-194)
- 문제: 사용자가 준비되지 않았을 수 있음 (충분한 피드백 없음)
- **개선**: 명시적인 "시작하기" 버튼으로 전환 제어

## 2. 정보 구조 (Information Architecture)

### 2.1 설정 메뉴 - 구조는 좋으나 정보 계층이 혼재

#### 현재 구조
```
설정
├─ AI 성격 설정
│  ├─ [아코디언] 커뮤니케이션 스타일
│  ├─ [아코디언] 성격/태도
│  ├─ [아코디언] 인지/분석
│  └─ [아코디언] 행동 방식
├─ 앱 설정
│  ├─ 다크 모드
│  ├─ 언어
│  └─ 선호 코드 언어
└─ 정보
   ├─ 버전 정보
   ├─ 시스템 정보
   └─ 도움말 링크
```

#### 문제점

**H. 아코디언의 첫 번째 그룹만 기본 열림**
- PersonaParameterGroup.tsx 라인 169: `defaultExpanded={index === 0}`
- 문제: 왜 첫 번째만 열려있는지 사용자 입장에서 불명확
- **개선 제안**: 모든 그룹을 처음엔 닫고, 아이콘으로 상태 명확하게 표시

**I. 설정과 AI 성격 설정의 구분이 애매함**
- "앱 설정"에는 테마, 언어만 있고
- "AI 성격 설정"에는 모두 슬라이더
- **문제**: 사용자가 어디서 뭔가를 찾아야 할지 불명확
- **개선**: 설정을 두 탭으로 나누기 (앱 설정 vs. AI 성격)

**J. 리셋 버튼의 위치와 위험성**
- Settings.tsx 라인 137: 헤더에 "초기화" 버튼
- 문제: confirm() 대화가 나오지만, 실수로 누르면 모든 커스터마이징 손실
- **개선 제안**: 리셋을 설정 페이지 하단에 별도 섹션으로 옮기기

## 4. 접근성 (Accessibility)

### 4.1 키보드 내비게이션

#### 현재 상태
- Cmd+K: 채팅 입력창 포커스
- Cmd+,: 설정 열기
- Cmd+Shift+S: 화면 추적 토글
- ESC: 포커스 해제 / 설정 닫기
- ?: 단축키 도움말

#### 문제점

**X. 누락된 키보드 단축키**
- Tab 키로 포커스 이동이 제대로 작동하는지 불명확
- 피드백 버튼(👍/👎/복사)에 대한 단축키 없음
- **개선**: 더 많은 키보드 단축키 추가

**Y. 대화 히스토리에서의 내비게이션**
- ConversationHistory: Enter로 대화 선택 가능한지 확인 불가
- **개선**: 키보드로 완전한 접근 가능성 보장

---

### 4.2 스크린 리더 지원

#### 현재 상태 (긍정적)
✅ `aria-label`, `aria-live`, `role` 속성 사용  
✅ 중요 요소에 대한 설명 제공  
예시: Chat.tsx 라인 339, 414-416

#### 문제점

**Z. 메시지 bubble의 구분**
- ChatBubble.tsx 라인 63-64: `role="article"`과 `aria-label` 사용 (좋음)
- 문제: 마크다운 렌더링된 내용이 스크린 리더에서 어떻게 읽히는지 불명확
- **개선**: MarkdownRenderer에서 시맨틱 HTML 확인 필요

**AA. 타임스탬프의 접근성**
- ChatBubble.tsx 라인 153: `aria-label`에 시간 포함
- 문제: "오후 2시 30분" vs "14:30" 표기 불일치
- **개선**: 일관된 형식 사용

---

### 4.3 색상 대비

#### 문제점

**BB. 타임스탐프 색상**

**CC. Muted 텍스트**

- 문제: 밝은 배경에서 가시성 부족할 수 있음

### 5.2 용어 일관성

**JJ. "화면 추적" vs "화면 분석"**

- Dynamic Island: "화면 추적"
- Chat.tsx: "화면 추적"
- **일관성**: 용어 통일됨 (✅)
